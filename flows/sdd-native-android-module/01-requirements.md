# 01-Requirements

## Overview

**react-native-replace-dialer** is a React Native library that enables Android applications to:
1. Check if the app is currently set as the default dialer
2. Request to become the default dialer app

## Stakeholders

- **Library consumers**: React Native developers building dialer/phone apps
- **End users**: Android users who want to use third-party dialer apps
- **Maintainers**: Library developers responsible for compatibility and bug fixes

## Functional Requirements

### FR1: Check Default Dialer Status

The library SHALL provide a method to check if the app is the default dialer:
- **Input**: Callback function
- **Output**: Boolean value via callback (true = is default, false = is not default)
- **Platform**: Android only
- **API Level**: Full functionality on Android M (API 23+)

### FR2: Request Default Dialer Status

The library SHALL provide a method to request becoming the default dialer:
- **Input**: Callback function
- **Output**: Boolean value via callback (true = success, false = failure)
- **Platform**: Android only
- **API Level**: Android M (API 23+)

### FR3: Native Module Validation

The library SHALL validate native module availability and provide detailed error messages with troubleshooting steps:
- Rebuild instructions
- CocoaPods instructions for iOS (future)
- Manual linking instructions
- Unit testing mock instructions

### FR4: Backward Compatibility

The library SHALL handle Android versions prior to API 23:
- Return `true` for `isDefaultDialer()` on pre-M Android
- Rationale: Default dialer concept not applicable to older versions

## Non-Functional Requirements

### NFR1: Performance

- Native module calls SHALL complete within 100ms for status checks
- Request operation time depends on system UI, not library-controlled

### NFR2: Compatibility

- **Minimum Android**: API 21 (Android 5.0 Lollipop)
- **Target Android**: API 34
- **Java Version**: Java 17
- **React Native**: >= 0.80.0

### NFR3: Architecture

- **Supported ABIs**: armeabi-v7a, x86
- **Pattern**: Callback-based async (not Promises)

## Known Issues

### ISSUE-001: Premature Callback Invocation

**Severity**: CRITICAL

The `setDefaultDialer()` method invokes the callback immediately with `true` before receiving the activity result from the system dialer selection UI.

**Impact**: 
- Callers cannot reliably determine if the user successfully set the app as default
- No way to handle cancellation or failure cases

**Root Cause**: 
- Incomplete implementation of `ActivityEventListener` interface
- Activity result handling code is commented out

**Resolution Required**: 
- Implement proper activity result handling
- Invoke callback only after receiving `RESULT_OK` or `RESULT_CANCELED`

## Out of Scope

- iOS support (currently Android only)
- UI components (library is headless)
- Permission handling (consumer app responsibility)
- Call management functionality (only dialer status management)
